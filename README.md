# OpenRouter API Wrapper

A robust and resilient Python FastAPI application that serves as a proxy/wrapper for OpenRouter LLM APIs. This service is designed to automatically handle rate limits (HTTP 429) and offline models (HTTP 404/5xx) by dynamically switching to other available free models.

## Key Features

- **Smart Model Fallback**: Automatically catches `429 Rate Limit` or `404 Not Found` errors. When triggered, it dynamically fetches the latest available models from OpenRouter and retries your prompt seamlessly.
- **Dynamic Caching**: Saves the fetched available models into a local `ready_models.json` file for auditing and debugging.
- **Out-of-the-box Swagger UI**: Interactive API documentation generated by FastAPI, allowing easy testing of endpoints.
- **Dockerized**: Fully containerized using `docker-compose` for rapid deployment and consistency.

## Setup Instructions

1. **Environment Setup**
   Ensure your `.env` file is properly configured with your OpenRouter API Key:
   ```env
   OPENROUTER_API_KEY=your_api_key_here
   FALLBACK_MODELS=google/gemma-2-9b-it:free,mistralai/mistral-7b-instruct:free
   DEFAULT_MODEL=google/gemma-2-9b-it:free
   ```

2. **Running the Service**
   Run the application using Docker Compose:
   ```bash
   docker compose up --build
   ```

3. **Accessing the API**
   - **Swagger UI**: [http://localhost:8000/docs](http://localhost:8000/docs)
   - **Chat Endpoint**: `POST /chat`
   - **Ready Models Endpoint**: `GET /models/ready`

## Architecture
The wrapper intercepts OpenAI-compatible Chat Completions requests, forwards them to OpenRouter, and manages failovers using `httpx` and `openai` Python SDKs.
